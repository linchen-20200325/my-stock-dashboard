## 1. 系統架構總覽

### 1.1 設計理念

本系統採用「由上而下 → 由外而內」三層過濾架構：

```
Layer 1：總體面（宏觀）過濾
    ↓ 市場體制 × 宏觀警示燈
Layer 2：策略面（中觀）篩選
    ↓ V4 六模組 × 領先指標 × V5 大師模組
Layer 3：個股面（微觀）評分
    ↓ 多因子評分 × 風控模組
```

只有三層同時通過，才執行買入。任一層亮紅燈，強制降倉或停止操作。

## 2. 市場體制引擎

**檔案**：`market_strategy.py`　｜　**核心函數**：`market_regime()`

### 2.1 體制定義

| 體制 | 標誌 | 曝險上限 | 觸發條件 |
|---|---|---|---|
| 多頭 | 🟢 牛市 | **80%** | 評分 ≥ 3 |
| 中性 | 🟡 震盪 | **50%** | 評分 = 2 |
| 空頭 | 🔴 熊市 | **20%** | 評分 < 2 |

### 2.2 五大評分指標

每項指標加分，滿分 **6.5 分**，門檻 3 分（多頭）。

#### 指標一：站穩 MA60 三日確認（+1.5 分）

> **三日確認規則（Hysteresis 遲滯區間）**：連續三個交易日收盤均高於 MA60，才計分為「確認突破」；同樣，連三日跌破才計入「確認轉弱」。尚未滿三日時維持前一狀態，不加分也不扣分。此設計防止大盤在季線附近盤整時頻繁加減倉（「雙巴」耗損）。

| 條件 | 得分 |
|---|---|
| **連三日**收盤 > MA60（確認站穩） | +1.0 |
| 且 MA60 本身上彎（當日 > 昨日） | +0.5 |
| 未滿三日（過渡狀態） | 0（不加不扣） |
| **連三日**收盤 < MA60（確認跌破） | 不得分 |

#### 指標二：站穩 MA120 三日確認（+1.5 分）

> **三日確認規則**：連續三個交易日收盤均高於 MA120，才算確認。

| 條件 | 得分 |
|---|---|
| 連三日收盤 > MA120 | +1.0 |
| 且 MA120 上彎 | +0.5 |

#### 指標三：外資買超（+1 分）

| 條件 | 得分 |
|---|---|
| 外資現貨淨買超 > 0 | +1.0 |

**資料來源**：TWSE OpenAPI（當日三大法人買賣超）

#### 指標四：漲跌比（+1 分）

| 條件 | 得分 |
|---|---|
| 上漲家數 / 下跌家數 > 1.0 | +1.0 |

#### 指標五：M1B-M2 貨幣利差（+1.5 分）

| 條件 | 得分 |
|---|---|
| M1B 年增率 - M2 年增率 > 0 | +1.0 |
| 且利差持續擴大（趨勢向上） | +0.5 |

> M1B 代表活存資金、M2 代表整體貨幣供給。利差轉正代表資金從定存流向股市，為強烈領先指標。

### 2.3 量能燈號（附加過濾）

| 量能狀態 | 條件 | 解讀 |
|---|---|---|
| 爆量追高警示 | 成交量 > 20日均量 × 1.3 | 恐慌性追買，體制強化但需注意反轉 |
| 縮量盤整 | 成交量 < 20日均量 × 0.7 | 方向不明，降低操作頻率 |

### 2.4 曝險比例計算

```python
def portfolio_exposure(regime):
    if regime == "bull":   return 0.80
    if regime == "neutral": return 0.50
    if regime == "bear":   return 0.20
```

## 3. V4 策略引擎

**檔案**：`v4_strategy_engine.py`　｜　**類別**：`V4StrategyEngine`

V4 共六個獨立模組，採「模組化否決制」：紅燈模組出現，對應操作禁止執行。

---

### 模組一：宏觀否決燈（Macro Veto）

**函數**：`check_macro_veto()`

| 燈號 | 條件（任一符合） | 最大倉位上限 |
|---|---|---|
| 🔴 紅燈（高風險） | VIX > 25 **或** 外資期貨淨口數 < -20,000 | **20%** |
| 🟡 黃燈（中風險） | VIX > 20 **或** 外資期貨淨口數 < -10,000 | **50%** |
| 🟢 綠燈（安全） | 其餘情況 | **100%** |

**判斷邏輯**：

```
VIX > 25  →  恐慌性拋售，市場流動性危機風險極高
VIX 20-25 →  波動放大警示，降低持倉等待
外資期貨大空 → 外資用期貨避險，代表機構看空現貨
```

---

### 模組二：相對籌碼比率（Relative Chips Ratio）

**函數**：`calc_relative_chips(days=5)`

**公式**：

```
外資籌碼比率 = Σ(5日外資淨買) / 流通在外股數 × 100%
投信籌碼比率 = Σ(5日投信淨買) / 流通在外股數 × 100%
```

**訊號判斷**：

| 訊號 | 條件 | 意義 |
|---|---|---|
| 🔴 強力集中 | 外資比率 > 0.5% **或** 投信比率 > 0.3% | 機構大量吸籌，強烈看多 |
| ⚪ 中性 | 介於集中與分散之間 | 觀望等待 |
| 🟢 分散出貨 | 外資比率 < -0.5% **或** 投信比率 < -0.3% | 機構連續出貨，規避風險 |
| ✅ 連買加分 | 連續三日以上淨買超 | 趨勢更為確認，加強訊號 |

---

### 模組三：頭部壓力（VPOC Overhead Resistance）

**函數**：`find_overhead_resistance(lookback=120)`

**算法**：成交量加權价格分布（VPOC = Volume Point of Control）

1. 取近 120 日高低價範圍，切成 **20 個等距價格桶**
2. 計算每個桶的成交量總和
3. 成交量最大的桶中心點 = VPOC（歷史主力成本區）

**距離計算**：

```
distance_pct = (VPOC - 當前價格) / 當前價格 × 100%
```

**壓力判斷**：

| 狀態 | 條件 | 操作建議 |
|---|---|---|
| 🔴 壓力存在 | 當前價 < VPOC 且距離 < 15% | 等待突破壓力再進場，或縮小部位 |
| 🟢 壓力解除 | 距離 > 15% 或 當前價已在 VPOC 以上 | 壓力消除，可正常操作 |

> VPOC 代表過去套牢者的平均成本，突破後才是真正解套，潛在賣壓消失。

---

### 模組四：停損防線（Stop Loss Level）

**函數**：`calculate_stop_loss()`

**計算邏輯**：

```python
# 找出近10日「量漲型紅K」（成交量 > 20日均量 × 1.5 且 收盤 > 開盤）
breakout_low = min(這些紅K的最低價)

# 停損線取最保守值
stop_loss = min(MA20, breakout_low × 0.98)

# 若無量漲紅K，fallback
stop_loss = 近10日最低點 × 0.98
```

**輸出指標**：

| 指標 | 說明 |
|---|---|
| `stop_loss` | 最終建議停損價 |
| `ma20` | 20 日均線（支撐參考） |
| `breakout_low` | 突破量紅K的最低點（量能底部） |
| `risk_pct` | 距當前價格的跌幅百分比 |

---

### 模組五：VCP 突破（Volatility Contraction Pattern）

**函數**：`detect_vcp_breakout(lookback_days=60)`

VCP 是 Mark Minervini 提出的波動收縮型態，代表主力完成洗盤、蓄勢待發。

**四個確認條件**（全部符合才觸發 BUY）：

#### 條件一：三階段波動率遞減

```
將60日分成3段，每段計算波動率：
波動率 = (段內最高 - 段內最低) / 段內最高

必須：V1 > V2 > V3（波動率逐段收縮）
```

#### 條件二：縮量窒息（最近5日）

```
最近5日平均量 < 50日均量 × 50%
```

> 縮量代表浮籌洗乾淨，主力持有，無人願意賣出。

#### 條件三：突破整理區高點

```
最新收盤 > 整理區高點 × 0.99（允許 1% 誤差）
```

#### 條件四：突破量爆量確認

```
突破當日成交量 > 50日均量 × 1.5
```

**價格目標（蔡森距離法）**：

```
target_price = 壓力區頂部 + (壓力區頂部 - 底部支撐)
```

即：目標漲幅 = 整理區高度，形成「測量移動」。

**訊號輸出**：

| 訊號 | 條件 |
|---|---|
| ✅ BUY | 四個條件全部符合 |
| ⏳ HOLD | 部分條件符合，等待確認 |
| ➖ NONE | 型態未成立 |

---

### 模組六：假突破偵測（False Breakout Detection）

**函數**：`detect_false_breakout_v4()`

**三個條件同時成立 → SELL 警示**：

| 條件 | 詳細說明 |
|---|---|
| ① 創 20 日新高 | 當日最高價為近 20 日最高點 |
| ② 爆量（天量） | 當日成交量 ≥ 近 60 日最大量 × 90% |
| ③ 醜K線 | 收盤 < 開盤（黑K）**或** 上影線 > 實體 × 2 |

**邏輯解讀**：

- 創新高 + 爆天量：吸引散戶追買
- 黑K / 長上影：主力在高點出貨，尾盤打壓
- 三者同時出現 = 典型「轎夫型態」，散戶被騙進去抬轎

## 4. 領先指標系統

**檔案**：`leading_indicators.py`

### 4.1 外資期貨淨口數

**函數**：`finmind_fut_oi()`

**公式**：

```
外資期貨淨口數 = TX 淨口數 × 1.0 + MTX 淨口數 × 0.25
```

| 數值區間 | 解讀 |
|---|---|
| > +20,000 口 | 外資大力做多，強烈看多訊號 |
| -10,000 ~ +10,000 | 中性，觀望 |
| < -10,000 口 | 外資開始避險 |
| < -20,000 口 | 外資大力做空，宏觀紅燈觸發 |

**備援機制**：FinMind API 失敗時改用 TAIFEX 公開資料 GET 請求。

### 4.2 外資選擇權淨口數

**函數**：`taifex_calls_puts_day()`

**公式**：

```
選擇權淨口數 = (BC - SC - BP + SP) / 10

BC = 買方買進 Call 未平倉口數
SC = 賣方賣出 Call 未平倉口數（外資收 Call 保費）
BP = 買方買進 Put 未平倉口數
SP = 賣方賣出 Put 未平倉口數（外資收 Put 保費）
```

> 外資傾向賣出選擇權（收保費），因此**正值**代表外資做多、**負值**代表做空。

### 4.3 韭菜指數（散戶情緒逆指標）

**函數**：`taifex_mtx_data()`

**公式**：

```
韭菜指數 = (三大法人 MTX 空單 - 多單) / 全部 MTX 未平倉 × 100
```

| 數值 | 意義 | 操作含義 |
|---|---|---|
| 正值（散戶淨多） | 散戶大量做多 | ⚠️ 危險！散戶追高，考慮減倉 |
| 負值（散戶淨空） | 散戶大量做空 | ✅ 機會！散戶恐慌，考慮進場 |

> 韭菜指數為**逆向指標**，散戶情緒極端時往往是反轉訊號。

### 4.4 Put/Call 比率（PCR）

**函數**：`taifex_pcr()`

**公式**：

```
PCR = Put 未平倉口數 / Call 未平倉口數
```

| 數值 | 市場情緒 | 解讀 |
|---|---|---|
| < 0.80 | 樂觀（Call 主導） | 市場過度樂觀，注意頂部 |
| 0.80 ~ 1.20 | 中性 | 多空均衡，觀望 |
| > 1.20 | 悲觀（Put 主導） | 市場恐慌，可能為底部機會 |
| > 1.50 | 極度恐慌 | 🚨 宏觀紅燈觸發 |
| < 0.50 | 極度樂觀 | 🚨 宏觀紅燈觸發 |

### 4.5 三大法人現貨買賣超

**函數**：`twse_institutional_day()`

| 法人 | 特性 | 解讀重點 |
|---|---|---|
| 外資 | 全球資金流動，趨勢性強 | 連續買超 > 5 日，趨勢確認 |
| 投信 | 季底作帳習慣，本土資金 | 季末最後兩週投信護盤常見 |
| 自營商 | 短線避險操作 | 可參考但雜訊較多 |

### 4.6 融資融券（信用市場）

| 指標 | 意義 |
|---|---|
| 融資餘額上升 | 散戶加槓桿追高，風險增加 |
| 融資餘額下降 | 散戶去槓桿，恐慌出場，可能是底部 |
| 融券餘額上升 | 空頭力道增加，但也是軋空燃料 |

### 4.7 前五大 / 前十大交易人未平倉

**函數**：`taifex_large_trader()`

| 指標 | 意義 |
|---|---|
| 前五大淨多單 | 主力大戶看多，訊號較強 |
| 前五大淨空單 | 主力大戶避險或看空 |

## 5. V5 大師模組

**檔案**：`v5_modules.py`

---

### 模組五：基本面領先指標（合約負債 + 資本支出）

**函數**：`analyze_fundamental_leading()`

**龍多股條件**（兩者同時滿足）：

| 條件 | 閾值 | 意義 |
|---|---|---|
| 合約負債 YoY 成長 | > 20% | 預收款項大增，代表訂單爆滿 |
| 資本支出 / 實收資本額 | > 80% | 公司大幅擴產，確信未來需求 |

**訊號輸出**：

| 燈號 | 條件 | 意義 |
|---|---|---|
| 🔴 龍多股 | 兩條件皆符合 | 基本面強力支撐，優先關注 |
| 🟡 部分符合 | 僅一條件符合 | 繼續觀察 |
| ⚪ 普通 | 皆不符合 | 無特殊基本面加分 |

### 模組六：相對強度（RS Relative Strength）

**函數**：`calc_relative_strength()`

**公式**：

```
RS = (個股N日報酬率 - 大盤N日報酬率) / 大盤N日波動率

計算週期：20日、60日、120日（三個時間框架 Z-score 標準化）
```

**訊號判斷**：

| RS 值 | 燈號 | 意義 |
|---|---|---|
| ≥ 1.0 | 🔴 逆勢強股 | 大盤跌時個股仍強，主力護盤 |
| 0.3 ~ 1.0 | 🟡 偏強 | 略優於大盤 |
| -0.3 ~ 0.3 | ⚪ 同步大盤 | 跟隨大盤，無特殊 |
| < -0.3 | 🟢 落後大盤 | 弱勢股，避免操作 |

> **RS 使用方式**：優先選擇 RS ≥ 1.0 的股票，尤其是大盤下跌時個股不跌或上漲，代表主力在保護價格。

### 模組七：本益比河流（Valuation River）

**函數**：`calc_valuation_zone()`

**計算方式**：以歷史 PE / PB 的均值 (μ) 和標準差 (σ) 劃定五個估值區間。

**估值區間**：

| 區間 | 條件 | 燈號 | 操作建議 |
|---|---|---|---|
| 特價區 | PE < μ - 2σ | 🟢🟢 | 強力買入機會 |
| 便宜區 | μ-2σ ≤ PE < μ-σ | 🟢 | 可積極布局 |
| 合理區 | μ-σ ≤ PE ≤ μ+σ | ⚪ | 正常持有 |
| 昂貴區 | μ+σ < PE ≤ μ+2σ | 🔴 | 降低買入意願 |
| 超貴區 | PE > μ+2σ | 🔴🔴 | 停止買入，考慮減倉 |

**特殊處理**：EPS < 0 時改用 PB 河流，避免負本益比誤判。

### 模組九：布林帶突破（Bollinger Breakout）

### 7.2 ATR 動態停損（Dynamic Stop — 主規則）

固定百分比停損無法適應不同波動率的標的：高波動股用 -8% 過窄，低波動股用 -8% 過寬。
v4.0 起全面改用 **ATR（Average True Range）動態停損**：

$$\text{Stop Loss} = \text{Buy Price} - \text{ATR}_{14} \times 1.5$$

```python
# risk_control.py — atr_stop_price()
def atr_stop_price(buy_price, atr, multiplier=1.5, fallback_pct=0.08):
    if atr is not None and atr > 0:
        return round(buy_price - atr * multiplier, 2)   # 動態停損
    return round(buy_price * (1 - fallback_pct), 2)     # 安全備援 -8%
```

**觸發邏輯**：當現價跌破 `atr_stop_price(buy_price, atr14)` → 立即停損出場。

**適應性說明**：
- 高波動標的（ATR 大）→ 停損距離自動拉開，不被正常回檔震出
- 低波動標的（ATR 小）→ 停損收緊，快速止損保護資金
- ATR 未計算（資料不足）→ 自動降級為固定 -8% 安全備援
